    AEP: 5
    Title: Origin mirrors
    Author: Alejandro R. Mosteo <amosteo@unizar.es>
    Status: Draft
    Created: 2026-06-13

Abstract
========

Implement support for multiple origins, so when the main one is down, alr can
use a mirror to fetch a release.

Rationale
=========

As of this document, alr supports a single origin under the `[origin]` object
in an index manifest. The proposal is to allow this origin to be an array
(`[[origin]]`), with the first one being the authoritative source (the origin)
and the rest being mirrors. This way, no need types are introduced, although
some differences between the first origin and the following ones are described.

For the first implementation, mirrors must be of the same exact kind as the
origin, and contain a bitwise copy. That is, there cannot be hashes or commits,
and if there are, the should match the origin's. This could be relaxed in the
future perhaps, but maximum safety and coherence is the goal for now.

New, special origins (e.g., Software Heritage Foundation) could be added in the
future that would not need to adhere to this rule, as they use their own
storage format.

A new optional field, `name`, will be added that will allow selecting a mirror
or even give a list of priorities. Until this is implemented, the release would
be fetched using the origin and mirrors in order of appearance.

Implementation steps
--------------------

Progressive implementation through several PRs.

1. Make Origin into optionally an array. Always create it as an array during
   publication, to simplify latter addition of mirrors. Test loading of
   releases with multiple origins, although they are discarded. Implement
   verifications during loading: either no hash/commit, or matching origin's.

1. Do use mirrors in array order after origin failure.

1. Add `name` field. Make the name `origin` reserved, and use it for the first
   origin in the array when not given. Add a setting to give a comma-separated
   list to try mirrors in order of preference. Implement the fetching in order.
